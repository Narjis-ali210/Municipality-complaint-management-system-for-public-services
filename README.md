 🏛️ e-Municipality: Complaint Management System for Public Service Issues

**A Graduation Project Presentation Submitted in Partial Fulfillment of the Requirements for the Degree of Bachelor in Computer Science & Information Technology – University of Karbala (2025 - 2026)**.


### 📝 Project Overview

In the era of rapid digital transformation, **e-Municipality** is an integrated, mobile-based platform designed to revolutionize and modernize how citizen complaints regarding public services and infrastructure are handled. The system aims to phase out slow, paper-based, and non-transparent traditional methods by replacing them with a smart, cloud-powered solution that fosters direct communication between citizens and municipal authorities.


### ⚙️ Problem Statement & Research Aim

* **The Problem:** Traditional complaint management channels are plagued by manual processing delays, lack of transparency, and a complete absence of tracking mechanisms. Citizens suffer from unaddressed infrastructure failures (e.g., waste management, utility breakdowns, road repairs) with no way to follow up.

* **The Solution (Aim):** To design and implement a secure, cross-platform mobile application connected to a centralized cloud database and an administrative dashboard to manage, review, and resolve public complaints in real-time.

---

### 👥 System Architecture & Core Features

The system operates on a seamless Client-Server architecture with two dedicated role-based interfaces:

#### 1. Citizen Mobile Application

* **Secure Authentication:** Easy user registration and token-based secure login.


* **Smart Complaint Submission:** Allows users to submit detailed complaints including title, category, description, and optional image attachments.


* **Location-Based Reporting:** Integrated with maps to pinpoint the exact location of the issue using Latitude and Longitude.


* **Real-Time Tracking:** Citizens can track their complaint status through live stages: **`Pending` ➡️ `In Progress` ➡️ `Resolved**`.



#### 2. Administrative Dashboard

* **Centralized Overview:** Displays quick analytical statistics on new, open, and closed complaints submitted over the last 24 hours.


* **Advanced Management:** Allows admins to filter complaints by specialized ministry/department or geographical region.


* **Live Status Synchronization:** Admins can quickly update complaint statuses and respond directly to citizens, instantly triggering real-time user notifications.

---

### 💻 Tech Stack & Development Tools

Built using modern, highly scalable, and high-performance industry-standard technologies:

* **Frontend Framework:** **Flutter** (Dart Language) – chosen for its rich widget library, high performance, and responsive UI design across cross-platform screens.


* **Backend Cloud Services:** **Firebase Platform**:


* **Firebase Authentication:** Handles secure role-based access control.


* **Cloud Firestore:** A NoSQL database powering live, instant data synchronization (Real-time Sync).


* **Firebase Storage:** Provides secure storage and retrieval for image attachments.




* **Development Environment:** VS Code / Android Studio & Git version control.
